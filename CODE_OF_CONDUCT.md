# Code of Conduct

## Our Standard

Halonyx is a small student project. The bar is simple: be someone people want to collaborate with.

That means:

- Give feedback on the code, not the person. "This has a timing side-channel" is useful. "This is amateur" is not.
- Assume good faith. Most bad PRs are inexperience, not malice.
- Disagree openly, but back it up — especially on protocol/crypto decisions, where "I think" needs a "because."
- Respect that this is maintained by students alongside coursework. A slow response isn't a snub.
- No harassment, personal attacks, discriminatory language, or sexualized content, in issues, PRs, commits, or anywhere else connected to this project.
- Don't publicly disclose security vulnerabilities — see [`SECURITY.md`](SECURITY.md) for how to report them properly instead of turning an issue thread into a live exploit demo.

## Scope

Applies to this repository — issues, pull requests, commits, discussions — and to any other space where you're representing the project (e.g. reaching out to a maintainer directly about it).

## Enforcement

Violations can be reported by emailing the maintainer directly or opening a private report via GitHub. Reports are handled quietly and taken seriously; retaliation against someone for reporting in good faith is itself a violation.

Consequences scale with severity and pattern — a first-time sharp comment gets a note, not a ban. Sustained harassment, doxxing, or bad-faith disruption gets you removed from the project: comments hidden, PRs closed, and blocked from further participation.
